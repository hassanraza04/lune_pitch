Here's a take on how the website could express it as confidently as the product does.

Tour the redesign See the original Read the proposal
Why this redesign

The product is enterprise-grade. The site should match.

A few observations, framed honestly. Nothing here is criticism of the current build. These are opportunities I'd want to invest in if I were on your team.

Today

  • Hero communicates the platform, but doesn't quickly land the headline metrics
  • The four key features (Multi-Lingual, Future-ready, Modular, Track-record) are mentioned but not visually anchored
  • Case studies list named clients without a clear at-a-glance hierarchy
  • "Schedule a Demo" is the only primary action; no secondary path for developers
  • Mobile and dark-mode polish has room to grow

This concept

  • Hero leads with three real stats animated on entry (1.5B+, 0.5ms, 99%)
  • Four key features become a dedicated section between Platform and Industries
  • Case study cards use imagery + tag + name with hover affordances
  • Two CTAs: "Schedule a Demo" for sales, "Explore products" for technical buyers
  • Dark-first design tuned to the mint accent in the logo; fully responsive
What I built

Not just a design. A working stack.

18 pages, one design system, live deployment on Cloudflare Pages, and a real backend behind the contact form. Everything below is shipped at lunepitch.pages.dev right now.

Design system

One ~50 KB stylesheet, custom-property tokens sourced from your logo. Responsive grid, fluid type, dark-first.

A+ security headers

HSTS preload, strict CSP, X-Frame-Options DENY, Permissions-Policy locked down. Verifiable on securityheaders.com.

Live contact backend

Cloudflare Pages Function. Server validation, Cloudflare Turnstile, IP rate limit, honeypot. Resend delivers to inbox.

Cookie consent + analytics

Vanilla-JS banner with category toggles. Cloudflare Web Analytics beacon only loads after opt-in. No tracking by default.

Full SEO + social previews

Per-page canonical, Open Graph, Twitter Card, JSON-LD (Organization + WebSite + per-page schema). Sitemap, robots.txt, security.txt.

Developer docs site

Sidebar nav, 5 endpoint pages, language-switching code samples (cURL / Python / Node / Go), parameter tables.

Audit folder

Documented security headers, a11y checklist, Lighthouse instructions, pen-test scope. A static-scan script ships with the repo.

Tasteful motion

Scroll-reveal IntersectionObserver, animated stat counters, sticky blurred nav. prefers-reduced-motion respected.

Noted from the latest design pass

What I'd want to add, with you.

A recent review surfaced specific items that need either real brand assets, real product data, or a feature build. Logged here so they don't get lost.

Needs product asset

Animated enrichment in the hero

A short loop showing a raw transaction string becoming structured data: category, merchant logo, location, carbon. Replaces decorative geometry with a demonstration of the product itself.

Needs API access

Live "try Enrich" sandbox

Paste a transaction string above the fold, get the live enriched response from a sandboxed test key. The strongest possible self-serve qualifier for technical buyers.

Needs brand files

Real customer & certification logos

Swap the text-pill "Trusted by" strip for actual customer logos (ADIB, NBF, D360, Beyon Money, Alaan Pay), and use the official Dubai AI Seal, SOC, and ISO 27001 marks in the Certified By block.

Needs real numbers

Outcome metrics on every case-study card

One quantified result per customer story (engagement uplift, time-to-deploy, transactions processed). The 98% categorization accuracy stat from the ADIB story is already pulled into the home page. The rest need to come from Lune's own deployment data.

Needs integration

Calendar booking on contact

Embed Cal.com or Calendly directly on the contact page so prospects can book a 20-min walkthrough without waiting for an email reply. The form stays as the secondary path.

Feature build

⌘K command palette & site search

Global search across docs, case studies, and product pages. The kind of nav cue that signals "this is a serious developer product." Linear and Vercel both use this. About 1-2 days of work on top of the current build.

Beyond this static demo

What I'd build next, if we work together.

This static site is a starting point. Here's where I'd take it.

Phase 1 · Content

CMS & editorial workflow

Wire blog and case-study pages to a headless CMS (Sanity, Contentful, Hygraph). Editors update copy without code; preview branches for stakeholder review before publish.

Phase 1 · Content

Real legal & cookie policy

Replace the placeholder Privacy / Terms / Cookie copy with your counsel-reviewed text. The infrastructure (consent banner, opt-in gating, preference modal) is already in place.

Phase 1 · Content

Brand & illustration

Replace Unsplash stock photography with custom product visualizations (UI mocks, data-flow diagrams, branded illustration of the merchant graph).

Phase 2 · Conversion

Industry-specific landings

Dedicated pages per vertical (Retail Banks, Neo-Banks & Wallets, Fintechs), each with their own social proof, KPIs, and CTA. Drive paid traffic to focused pages.

Phase 2 · Conversion

Interactive Enrich playground

Live, in-browser tool: paste a transaction string, see the enriched response from a sandboxed test key. Self-serve qualification for developers.

Phase 2 · Conversion

Arabic / RTL variant

Arabic-first variant of the site (full RTL layout, IBM Plex Sans Arabic). Most of your customers are in MENA — the site should meet them where they are.

Phase 3 · Scale

Custom domain & email

Point Lune's own domain at the deploy, set up SPF / DKIM / DMARC, switch the contact form to send from a real Lune address instead of onboarding@resend.dev.

Phase 3 · Scale

External pen test

External firm runs a real penetration test before any production customer onboards. Scope already documented in audits/penetration-test-scope.md.

Phase 3 · Scale

Continuous monitoring

Error tracking (Sentry), uptime monitoring (Better Stack), Lighthouse budgets in CI, dependency scanning. The static-scan script in this repo is the start.
